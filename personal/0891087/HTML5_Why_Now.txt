주제 : HTML5 가 무엇인가, 왜 출현했는가, 현재상태와 미래상태 주요기능과 브러우저 지원 상태는?

  HTML5란, HTML의 차기 주요 제안 버전으로 월드 와이드 웹의 핵심 마크업 언어이다. 2004년 6월 Web Hypertext Application Technology Working Group(WHATWG)에서 웹 애플리케이션 1.0이라는 이름으로 개발을 시작하였으며, 2010년 3월 WHATWG의 초안 표준 상태(Draft Standard state)이고, W3C에서는 작업 초안 상태이다. HTML5는 HTML 4.01, XHTML 1.0, DOM Level2 HTML에 대한 차기 표준 제안이다.
 HTML5의 출현 배경은 처음 W3C에서 1998년에 HTML4가 발표되고 그 뒤에 XHTML1.0이 발표되었고 XHTML2.0을 개발하였지만 실패로 돌아가 대부분의 브라우저들이 받아들이지 못하고 원래 속해있던 W3C를 떠나게 되었으며, 떠난 브라우저들이 뭉처서 WHATWG라는 협회를 만들고 새로운 언어를 만들어 쉽고 정확하며 편하게 개발을 하자라는 취지로 HTML5를 선보이게 된다. 결론적으로 왜 HTML5가 나왔냐면 여러 브라우저들 끼리 호환성이 높게 만들자라는 것과 여러가지의 마크업과 그 마크업으로 인해 개발자들이 편하게 개발할 수 있는 환경을 제공해 줌으로써 개발자들의 개발 시간도 줄어드는 효과를 얻을 수 있으므로 생산성이 더욱 높아지는 결과를 가져다 줄것이라는 기대감 때문에 출현하였다고 본다. 그 기대감으로 만들어진 HTML5는 이제 거의 모든 브라우저의 기본 틀로 변해가고 있는 상황이다.
  조사한 후 생각해 보았더니 W3C라는 곳은 약간 박쥐같은 습성이 있는것 같다. 처음에 HTML을 개발하고 시장에 내 놓았고 버전을 업그레이드 하였지만 브라우저들 (크롬, 오페라, IE, FireFox, Safari 등)이 거부하면서 이런 브라우저들이 나와서 개발을 하여 선보인것이 지금의 HTML5의 처음만들어진 배경이 된다. 사람들이 많은 호응을 관심을 보이기 시작한 결과 W3C는 독립한 브라우저들을 무시하다가 현재에 이르러 독립된 브라우저 협회(WHATWG)와 손을 잡고 다시 HTML5라는것을 개발하고 있는 상황이다. 이를 비추어 봤을때 W3C는 좋은쪽으로만 붙으려는 경향이 보이는 생각이 들었다. 
  조사를 통해 또한가지 알아낸 사실이 있다. 바로 W3C에서 만든 HTML을 거부한 브라우저들중 IE 같은 경우 처음부터 다른 브라우저(크롬, 오페라 등)들과 HTML5를 개발한것은 아니었다는 사실이다. IE도 물론 W3C를 거부하였지만 그렇다고 HTML5를 사용하지 않았다. 하지만 현재 HTML5의 뛰어난 성능으로 인해 IE도 HTML5의 개발과 습득을 하고 있는 것은 사실이다. IE는 다른 브라우저들에 비해 늦게 HTML5라는 곳으로 뛰어들어서 IE10버전에 많은것을 HTML5로 맞추고 있지만 결국 다른 브라우저들에 비해 HTML5지원은 늦고 있다는것은 사실이다. 세계적으로 보았을때 아직 IE의 비율은 상당히 높은 편에 속한다. 이를 미루어 봤을때 IE도 다른 브라우저들과 같이 HTML5지원을 조속히 더 많이 해야할 필요가 있다고 생각된다.
  현재 HTML5의 상태는 초안상태이기는 하지만 많은 브라우저 업체들이 이미 HTML5기능들을 지원하고 있다.
  구글 및 애플 닷컴 사이트의 첫 소스코드는 HTML5 Doctype을 이용하고 있다. 즉, 지금 웹 사이트에 HTML5 마크업을 이용할 수 있다. 물론 IE와 같이 오래된 브라우저의 경우 호환 스크립트로 기능 추가(Fallback)을 하는 방법을 사용하면 된다. HTML5 Shive, Modnizer, IE-CSS3 같은 스크립트를 통해 하위 호환성을 보장해 줄 수 있다. 단, HTML5를 통한 웹 사이트 제작이 회사의 개발 생산성을 높히고 고객의 요구와 비지니스에 부합한다는 전제하에 말이다. 미래 웹 기술을 미리 선도적으로 도입함으로서 부가 이익도 누릴 수 있다. 2004~2005년도에 웹 표준 기반 CSS 레이아웃 도입에 많은 토론이 있었지만 현재 결국 많이 도입되어 있다.
  앞으로 HTML5의 상태는 여러 브라우저들이 HTML5를 지향함에 따라 접할 기회가 많아지고 또 브라우저 뿐만 아니라 모바일까지도 HTML5의 영향력이 더운 커져서 새로운 웹 기술이기 때문에 학습 비용이 많이 들어도 HTML5의 기술을 배우는것이 필수가 될것이다. 또한 HTML5로 인한 업무 분담이 확실해 져서 개인의 영역이 분명해 지고 그 일에 매진할 수 있게 될 것이다. HTML5는 모바일에서도 그 기능을 발휘하는데 스마트폰이 대두화된 이시점에서 HTML5의 역할은 더더욱 증가될 것이다.
  주요기능에는 다음과 같다.
  - 웹폼 (Web Form) : 사용자의 입력정보를 받기 위해 사용되는 입력 형태에 대한 정의에 사용되는 마크업.
  - 캔버스 (Canvas) : 웹에서 즉시모드(immediate mode)로 2차원 그래픽을 기리기 위한 API와 canvas 내 각종 객체를 회전, 변환하고 그레디언트, 이미지 생성 등 각종 효과를 주는 기능에 대한 API 지원
  - SVG (Scalable Vector Graphic) : XML 기반의 2차원 벡터 그래픽을 표현하기 위한 언어
  - Video/Audio : video는 비디오 또는 영화를 보여주기 위해 사용되는 미디어 엘리먼트이며, audio는 사운드나 오디오 스트림을 표현하기 위한 미디어 엘리먼트
  - Geolocation : 디바이스의 지리적 위치 정보를 제공하는 API 표준 지원
  - 오프라인 웹 응용 (Offline Web Application) : 인터넷 연결이 지원되지 않는 경우에도 웹 응용이 정상적으로 수될 수 있도록 지원하는 기능으로 응용에 대한 캐싱과 데이터에 대한 캐싱으로 구성
  - 웹 SQL 데이터베이스 (Web SQL Database) : 다양한 표준 SQL을 사용해 질의할 수 있는 데이터베이스 기능에 대한 API 지원
  - 로컬저장소 (Local Storage) : 기존의 쿠키의 기능을 개선하기 위한 목적으로 개발된 기능으로 웹 클라이언트에서 키와 값이 쌍으로 구성된 데이터를 영구적으로 저장하는 기능
  - 웹 소캣 (Web Socket) : 웹 응용이 서버 측의 프로세스와 직접적인 양방향 통신을 위한 API 지원
  - 웹 워커 (Web worker) : 웹 응용을 위한 쓰레드(Thread) 기능에 대한 API 지원
  이와같은 기능을 구현하는 HTML5는 HTML4보다 확장된 태그를 지원한다. 또 문서 구조에 적합하게 구조화 마크업을 사용함으로써 개발자를 위한 것이 잘 나타나 있다. 또한 편리한 폼속성으로 개발자의 어려움을 덜어주는 역할도 한다. 적절한 속성값을 사용하면 쉽게 구현할 수 있도록 해주는 점을 봤을때 개발하는데도 시간을 현저히 줄여 줄 것으로 기대된다. 위에서 언급한것과 같이 HTML5는 다양한 API를 제공한다. 이를 리치 웹 애플리케이션이라 하는데 새로운 마크업 요소와 폼 속성들이 더해지면 더 좋은 애플리케이션 개발에 사용할 수 있다.
  2012년 4월 현재 HTML5를 완벽하게 지원하는 브라우저는 없다. 점차 HTML5를 지원하는 범위가 확대되고 있다. 현재 4월 지원하는 현황은 다음과 같다.
  구글 크롬 > 애플 사파리 5.2 > 오페라 12알파 > 모질라 파이어폭스 11.0 > 오페라 11.60 > 인터넷 익스플로러 10 > 인터넷 익스플로러9 순으로 HTML5를 지원하고 있다.
브라우저 지원 상태 현황은 다음 싸이트를 통해 실시간으로 알 수 있다. -http://html5test.com

* WHATWG : W3C에서 1998년 HTML4.01을 발표하고 그들이 XHTML, XHTML2.0을 만들었으나 대부분의 브라우저 벤더들 (IE, OPERA, FireFox, Safari등)이 거부하면서 W3C를 이탕하기 시작하고 그에따라 만들어진 그룹
* 마크업 언어 : hyper text markyp language의 약어로, 홈페이지에 보이는 것들을 동적으로 움직이는 것.
		문서에 포함된 문장, 표, 소리 등과 같은 문서의 내용에 대한 정보가 아니라 그 문장과 그림, 표가 어떻게 배치되고 글자는 어떤 크기와 모양을 기지며, 들여쓰기와 줄간격, 여백은 어느 정도인지에 대한 정모를 의미.
* DOM : 설계 최적화 모델 (Design Optimization Model), 문서 객체 모델 (Document Object Model)
* CERN : 1954년에 스위스에 설립된 유럽 연합 공동 지원의 입자 물리학 연구소로 World Wide Web(월드 와이드 웹)의 개념이 처음 만들어진 곳이다. CERN의 컴퓨터 정보 수집 및 제어 관계 연구원이던 영국 출신의 팀 버너스 리는 1989년 인터넷을 편리하게 사용할 수 있는 하이퍼 텍스트(hypertext) 프로젝트를 제안했다.
* IETF : 인터넷국제표준화기구 (Internet Engineering Task Force), 인터넷의 운영, 관리, 개발에 대해 협의하고 프로토콜과 구조적인 사안들을 분석하는 인터넷 표준화 작업기구로 망 설계자, 운영자, 업체, 연구자들에 의해 구성되었다.
