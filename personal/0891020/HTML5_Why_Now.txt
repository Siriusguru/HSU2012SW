HTML은 웹 페이지를 작성하기 위한 웹 프로그래밍 언어로 Hyper Text Markup Language의 약자이다. Hyper는 웹 페이지를 클릭했을 때 연결된 다른 페이지로 이동하는 역할을 하며, Markup은 인쇄 교정 시 사용하는 주석을 의미한다. 즉, HTML의 기본적 의미는 웹 페이지의 텍스트, DATA 등에 주석을 다는 작업에 사용되는 언어이다. HTML5는 역시 HTML과 같이 웹 페이지를 만들기 위한 웹 프로그래밍 언어이고, HTML의 차기 프로그래밍 언어 버전이다. HTML5는 active X를 설치하지 않고도 설치한 것과 같은 동일한 기능을 구현하며, Flash나, MS사의 Silverlight, 자바FX 없이도 웹 페이지에서 그래픽 효과를 나타내는 것이 가능하다.
HTML5의 등장배경에는 XML과 HTML의 개발실패에 있다. XML과 HTML를 결합한 확장형 언어인 XHTML1.0이 개발된 후 XHTML2.0을 개발하고자 하고자 하였으나 개발을 실패하였다. W3C(World Wide Web Consortium: 1994년에 창립된 인터넷 관련 국제 컨소시엄)에서 XML2.0 개발을 주도했다. 근데 이 언어는 HTML의 언어와는 별개로 전혀 새로운 웹 언어를 만들기 위한 언어였기 때문에, 이전에 존재하던 HTML과 호환이 이루어지지 않았기 때문에 XML2.0을 정착시키고자 한다면 이전에 HTML로 만들었던 웹 페이지에는 접근을 할 수 없는 상태가 발생이 될 것 이였다. 그래서 HTML4.01을 잇는 새로운 웹 프로그래밍언어를 개발하기 시작하였는데, 오페라, 파이어폭스 등의 웹브라우저 개발회사의 개발자들이 비공개적으로 개발에 착수하였고, 이 개발자 그룹명을 WHATWG(Web Hypertext Application Technology Working Group)라고 칭하였다. 이때 오페라의 개발자 Ian Hickson이 책임자로서 개발을 진행하였고, Ian Hickson이 구글로 가면서 구글에서 정식 개발에 착수 하였다. 이때 개발중인 언어를 Web Application 1.0이라고 칭하였다. W3C는 몇몇의 개인 개발자들이 개발해온 Web Application 1.0을 인정하지 않았지만, W3G에서 그 동안 개발을 해오던 XHTML2.0이 엉망이 되면서, 생각을 돌려 WHATWG의 web Application1.0을 인정하였다. 2006년 W3C는 개발자 그룹을 새로 결정하였고, WHATWG의 개발내용으로 삼고 개발하기로 했다. 2년뒤인 2008년 WHATWG의 web application1.0을 HTML5로 명명하였고, 2009년 XHTML2.0의 개발 포기를 선언하였다. 2008년 Working Draft State이며 2022년이나 되야 제안승인(Proposed Recommendation)될 것이다. 그러므로 HTML5는 아직 시험준비 버전이다.
그러나 웹 브라우저에서의 HTML5를 적용 시키기 위해 버전업을 하고 있으며, 지원현황을 보면 2012년 1월에 HTML5를 완벽히 지원하는 브라우저는 없지만. 점점 HTML5의 범위가 확대되어가고 있다. HTML5의 검사점수를 보면 크롬16.0은 373/475점, 모질라 파이어 폭스10.0은 332/475 사파리는 302/475 IE9는 141/475점(IE10은 306/475)점으로 구글 크롬은 8단계의 버전업을 함으로써 현재 HTML5이 가장 많이 지원되고 있고, 6단계의 버전업을 한 파이어 폭스가 그 다음으로 많이 HTML이 지원 되고 있다.
주요 기능(특징)은  웹에서 즉시모드로 2차원 그래픽을 그리고, canvas내의 각종 객체를 회전, 변환하는 기능에 대한 API를 제공한다. 그리고 XML기반의 2차원 벡터 그래픽표현을 위한 언어를 제공하며, 인터넷 연결이 지원되지 않는 경우에도 웹 응용이 정상적으로 수행될 수 있도록 지원한다. 기존의 쿠키의 기능을 개선하기 위해 , 웹 클라이언트에서 키와 값이 쌍으로 구성된 데이터를 영구적으로 저장하는 로컬 저장소를 제공한다.
