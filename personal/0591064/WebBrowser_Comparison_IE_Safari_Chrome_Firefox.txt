1. IE (v9)
(1) 장점 : 대부분의 PC사용자들에게 익숙한 인터페이스를 갖고 있다. 주소표시줄에 주소를 입력하면 주소를 다 적지 않아도 열어본 페이지 목록, 즐겨찾기를 구분지어 보여준다.
(2) 단점 : IE7,8이 아닌 IE9를 사용해서인지 딱히 단점을 느끼지 못했다. IE7,8 과는 다르게 IE9에서는 속도문제로 불편함을 느낀 적이 없으며 다른 브라우저와 비교해도 그 차이를 느끼기 어려웠다. 다른 브라우저들과의 속도 경쟁을 위해 9 버전에는 GPU를 이용한 ‘하드웨어 가속’ 방식과 자바스크립트 해석 엔진인 ‘차크라’ 가 추가된 점이 그 이유인 것 같다. 이러한 IE9를 windows7 환경에서만 이용할 수 있다는 점이 단점이 될 수도 있겠다.

2. Safari (v5)
(1) 장점 : IE9와 마찬가지로 주소표시줄에 주소를 입력하면 주소를 다 적지 않아도 ‘방문기록’과 ‘책갈피’를 구분지어 보여준다.
(2) 단점 : Safari 설치 시 함께 설치할 수 있는 ‘bonjour’를 설치하면 IE와 Safari 실행 속도가 느려진다.
브라우저 실행 속도가 가장 느린 것 같다.
Safari를 통해서 온라인 게임을 시도하였으나 액티브X가 지원되지 않아서 게임을 할 수 없었다.

3. Chrome (v17)
(1) 장점 : IE9와 속도차이를 크게 느끼진 못했으나 IE9는 가끔 페이지 이동 속도에 기복이 있는 반면에 Chrome은 비교적 균일한 속도를 보여줬다.
IE9를 통해서 즐겨찾기목록을 가져오면 IE9와 거의 같은 모습의 인터페이스를 보여준다.
해외사이트 접속 시 번역기능이 능동적으로 나타난다.
(2) 단점 : 열어본 페이지 목록과 즐겨찾기 목록의 구분이 비교적 어렵다.

4. Firefox (v11)
(1) 장점 : 브라우저 실행 속도가 가장 빠른 것 같다. 
(2) 단점 : 다른 브라우저들과는 다르게 주소표시줄에 주소를 입력할 때 자동완성기능이 없고 크롬과 마찬가지로 열어본 페이지 목록과 즐겨찾기 목록의 구분이 쉽지 않다. 또한 열어본 페이지 목록에 필요 이상의 기록이 남아있다. (ex : 네이버 메인페이지 우측하단에 있는 쇼핑 카테고리).
네 개의 브라우저 중 페이지 이동 속도가 가장 느리다는 느낌을 받았다.
